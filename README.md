云棋AI（CloudChuessAI-Nv1.1）

一、项目概述

云棋AI（CloudChessAI）是一个以Python语言开发、基于Tkinter图形界面框架的五子棋综合应用。项目版本Nv1.1，发布于2026年5月4日。

二、核心AI算法

2.1 棋盘表示与评估

棋盘大小为11×11，采用二维整型数组表示（0为空、1为黑棋/玩家、2为白棋/AI）。
位置评分策略：综合考虑棋子到棋盘中心的距离、主副对角线位置、星位权重等因素，为每个交叉点赋予基础分数。
棋型评分：系统对每个方向上的棋子连珠模式（活一至活四、眠三、冲四等）进行细致的评分，并引入对角线权重系数，使AI能够准确识别关键棋型。
连接性与自由度评估：通过分析落子点周围己方棋子数量和空位数量，增强AI对棋势发展的判断能力。
战略性加成：对于形成长连威胁或“双三”“四三”杀局的着法，给予额外的分数奖励。

2.2 搜索算法

AI采用Minimax搜索配合Alpha-Beta剪枝的经典组合：
搜索深度与用户选择的难度挂钩：难度系数5/7/9/10对应不同的搜索深度，深度越大棋力越强。
动态搜索宽度：在搜索过程中，根据当前深度动态调整候选着法的数量，兼顾搜索质量与响应速度。
启发式着法排序：先对所有合法着法进行快速的静态评分，将高评分着法优先搜索，有效提升剪枝效率。
攻防一体化：在评分时同时考虑攻击分数和防守分数，使AI既能积极进攻也能稳妥防守。

2.3 性能优化技术

为了在有限的时间内完成深度搜索，系统采用了多种优化手段：
Cython编译：核心算法模块使用Cython编写，将Python代码转换为C扩展，大幅提升了循环和数组访问的效率。
Zobrist哈希缓存：为每个棋盘状态生成唯一的哈希值，并将评估结果和着法列表缓存起来，避免重复计算相同局面。
着法范围剪枝：只考虑已有棋子周围3格范围内的空位作为候选着法，极大减少了搜索分支数量。
提前终止检测：在搜索过程中若发现直接获胜或必败着法，立即返回。

三、功能模块详解

3.1 用户账号系统

注册与登录：新用户需提供用户名、密码（必须同时包含字母和数字，长度≥5，无特殊符号）和确认密码。密码经过SHA-512加盐哈希后存入YQ_DataBase.json，即使数据文件泄露也无法还原原始密码。
登录保护：连续登录失败超过4次，程序自动关闭，防止暴力破解。
账号管理：登录后用户可以修改昵称、密码、上传自定义头像（自动裁剪为圆形），也可以选择注销账号。注销操作会永久删除该用户的所有个人数据（包括基本信息和历史对局记录），且不可恢复。
云币与等级：每次对局获胜可获得云币奖励（与难度挂钩），每累计100云币提升1级。等级还直接决定解锁残局挑战的数量。

3.2 对战模式

系统提供三种对战模式：
AI对战（玩家先手）：玩家执黑先行，AI执白后手。
AI对战（AI先手）：AI执黑先行，玩家执白后手。
双人对战：
线下模式：两位玩家在同一台设备上轮流落子。
联机主机模式：一方创建服务器，指定端口号，等待其他玩家连接。
联机客户端模式：另一方输入主机IP地址和端口号，主动发起连接。
联机对战基于TCP Socket实现，程序会在后台线程中持续监听对方落子消息，确保对战过程流畅不卡顿。

3.3 残局挑战系统

云棋AI内置了40个精心设计的残局，按照难度分为四个等级：
简单残局（#1～10）：需要2～3步连杀，等级1解锁。
中等残局（#11～25）：需要3～4步连杀，等级3解锁。
困难残局（#26～40）：需要4～5步连杀，等级5解锁。
大师残局（#41～50）：需要5～7步连杀，等级8解锁。


3.4 成就系统
系统内置了12个成就，涵盖对局数量、胜场、连胜、击败高难度AI、联机对局、残局破解、等级提升等各个方面。成就达成条件实时监测用户数据，一旦满足条件即弹出祝贺提示，并自动发放云币奖励。已解锁的成就在个人信息页面中会以绿色背景和✅标记显示。

成就列表包括：初出茅庐、百战老将、首胜之光、常胜将军、棋圣降世、破产边缘、AI克星、社交达人、残局高手、残局大师、十连胜、等级飞升。

3.5 历史记录与复盘

每一局对弈结束后，系统都会自动保存以下信息：
完整的棋盘局面（以字符串形式存储）；
对局日期和时间；
对局类型（AI、me、player1、player2、other）。
用户可以在“个人信息”→“历史记录”中按日期和对手类型筛选历史对局。双击某条记录，会弹出一个棋盘详情窗口，以图形化方式展示当时完整的落子情况，方便复盘分析。用户也可以选择删除单条记录或清空全部历史。

3.6 多语言支持

系统支持简体中文和英文两种界面语言，所有界面文本（包括提示信息、按钮文字、菜单项等）均集中存储在CCS_CEI.py文件中。用户点击语言切换按钮后，程序会销毁当前窗口并重新创建，实现完全的中英文无缝切换。

3.7 关于与技术支持

“关于”页面展示了程序的详细信息：版本号、AI算法说明、发布时间、前端/后端设计者、赞助商、参考文献等。用户双击“技术支持”或“玩法规则”文本可以查看具体的致谢名单和游戏规则说明。

四、游玩方法与操作指南


4.1 安装与运行
确保所有项目文件保持完整目录结构（即CloudChessAI-Nv1.1.exe、font/、images/、files/、SourceCode/位于同一级目录下）。
直接双击运行CloudChessAI-Nv1.1.exe。
程序启动时会显示一个带有进度条的启动界面，加载完成后自动进入登录/注册主界面。

4.2 注册与登录

首次使用需要点击“创建账号”按钮，填写用户名、密码和确认密码。
注册成功后返回登录界面，输入用户名和密码，点击“登录”。
注意：密码不能为空，且必须同时包含字母和数字，长度至少5位，不允许出现特殊符号。

4.3 开始对局

登录成功后进入游戏主界面：
在顶部选择AI难度（MNX1.0-5/7/9/10，数字越大难度越高）。
点击五个选项卡之一选择对战模式：
AI对战（您是先手）：玩家执黑先行。
AI对战（您是后手）：AI执黑先行。
双人对战：进入子菜单选择线下或联机模式。
个人信息：查看/编辑个人资料、历史记录、成就列表、残局挑战。
关于：查看版本信息和致谢。

点击“开始游戏”按钮，棋盘激活。
点击棋盘上黄色交叉点进行落子（黑棋●、白棋○）。

当一方连成五子时，程序弹出获胜提示并自动保存对局记录，随后可开始新的一局。

4.4 联机对战步骤

创建主机（服务端）：
选择“双人对战”→“联机对战（创建主机）”。
在弹出的对话框中输入端口号（如12345）。
等待对方连接，连接成功后即可开始对局。

加入游戏（客户端）：
选择“双人对战”→“联机对战（加入游戏）”。
在地址簿中双击已有联系人，或双击“（空）”行添加新的IP地址和端口号。
点击连接，成功后即可与主机方对弈。

4.5 残局挑战步骤

在主界面点击“个人信息”→“残局挑战”。
查看当前等级下已解锁的残局列表。
双击某个残局进入挑战界面，界面上方会显示当前该谁走棋。
按照五子棋规则落子，破解成功会收到提示并获得云币。

4.6 修改个人信息

点击“个人信息”→“账号管理”→“修改信息”。
可以修改头像（从本地选取图片，自动裁剪为圆形）、昵称和密码。
修改完成后程序会刷新界面并更新数据库。

五、技术栈

编程语言	Python 3.8
图形界面	Tkinter + PIL（Pillow）
AI加速	    Cython(c/python)
网络通信	Socket + threading
加密算法	hashlib（SHA-512）
数据存储	JSON文件
字体渲染	ImageFont（PIL）
